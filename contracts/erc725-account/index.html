LSP0 ERC725Account

The ERC725Account contract is basically an implementation for the LSP0-ERC725Account Standard, this contract could also use simultaneously the LSP3-UniversalProfile-Metadata Standard to form a UniversalProfile where you could attach information to the Profile like the name, pictures, owned assets, etc under a certain schema.

This ERC725Account contract could be used as an account system to be used by humans, machines, organizations, or even other smart contracts. It has all the functionalities that an EOA have and even more, starting from executing functions, transferring value, deploying smart contracts via execute function, verifying signatures via isValidSignature function, being notified of incoming assets via universalReceiver function and the ability to set any information on the account via setData function.

note

ERC725Account implementation also contains the methods from ERC173 and ERC165.

executeâ€‹

  function execute(
    uint256 operationType,
    address to,
    uint256 value,
    bytes memory data
  ) public payable returns (bytes memory result)

Executes a call on any other smart contracts, transfers value, or deploys a new smart contract.

Triggers the Executed event when a call is successfully executed using CALL/STATICCALL/DELEGATECALL operations.

Triggers the ContractCreated event when a smart contract is created using CREATE/CREATE2 operations.

note

It can only be called by the current owner of the contract.

Parameters:â€‹

NameTypeDescription
operationTypeuint256The operation to execute.
toaddressThe smart contract or address to interact with. to will be unused if a contract is created (operation 1 & 2).
valueuint256The desired value to transfer.
databytesThe call data (ABI of the function to execute) , or the contract data to deploy.

The operationType can execute the following operations:

  • 0 for CALL
  • 1 for CREATE
  • 2 for CREATE2
  • 3 for STATICCALL
  • 4 for DELEGATECALL

Return Values:â€‹

NameTypeDescription
resultbytesThe returned data of the called function, or the address of the contract created (operation 1 & 2).

universalReceiverâ€‹

  function universalReceiver(
    bytes32 typeId,
    bytes memory data
  ) public returns (bytes memory result)

Emits an event when it's succesfully executed.

Call the universalReceiverDelegate function in the universalReceiverDelegate (URD) contract, if the address of the URD was set as a value for the UniversalReceiverKey in the account key/value value store of the same contract implementing the universalReceiver function and if the URD contract has the LSP1UniversalReceiverDelegate Interface ID registred using ERC165.

The current implementation of the UniversalReceiverDelegate is found here.

Triggers the UniversalReceiver event when this function get executed successfully.

databytesThe data received.

NameTypeDescription
